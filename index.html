The delivery pipeline

1. Commit code

Developer commits code to a github repository. View GitHub

2. Compile & test

A build server compiles the code and runs unit tests. View TeamCity

3. Packaged & pushed

The application is packaged and pushed to Octopus Deploy. View Octopus Deploy

4. Auto deploy to test

The application is automatically deployed to a test environment. A suite of automated web tests are run against the application.

5. Deploy to UAT

The application is deployed to a UAT environment, for stakeholders to see new changes.

6. Promote to production

When the various stakeholders are happy, the application is prompted to the production environment.
